# Domain Docs

Layout: single-context.

- Root glossary: `CONTEXT.md`.
- ADRs: `docs/adr/`.
- If repo becomes multi-context, add `CONTEXT-MAP.md` and update this file.

Rules:

- Read `CONTEXT.md` before naming domain concepts.
- Keep `CONTEXT.md` glossary-only.
- Put hard-to-reverse architectural decisions in ADRs.
- Do not put implementation notes in glossary.
